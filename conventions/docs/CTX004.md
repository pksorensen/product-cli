# CTX004 — Single-responsibility module doc comment

Every Rust source file (other than `mod.rs`, `main.rs`, `lib.rs`) must:

1. Start with a `//!` module-level doc comment.
2. Have its first doc line **not** contain the word " and " (surrounded by
   spaces).

## Why

The "no `and`" rule is a forcing function for module decomposition. If a
file's one-line summary needs a conjunction to describe what it does, the
file is doing two things and should be split. The Slice + Adapter pattern
in `CLAUDE.md` already encourages this — CTX004 is the build-time
enforcement.

`mod.rs`, `main.rs`, and `lib.rs` are excluded because they are
intentionally *integrating* modules: their job is to wire submodules
together, which inherently spans multiple responsibilities.

See [ADR-0005](../adr/ADR-0005-fitness-checks.md) for the full rationale,
including why this rule was promoted from a bash script to an xtask
check.

## Examples

**Triggers CTX004:**

```rust
// src/parse_and_render.rs
//! Parse YAML and render the resulting graph.
```

```rust
// src/utils.rs                                  (missing module doc)
use std::path::PathBuf;
```

**Passes CTX004:**

```rust
//! YAML front-matter parser for artifact types.
```

```rust
//! Knowledge graph construction from parsed artifacts.
```

## Enforcement

- **Tier:** 3b (xtask + syn).
- **Implementation:** `xtask/src/checks/ctx004_single_responsibility.rs`.
- **AST inspection:** parses each file with `syn::parse_file`, walks the
  inner `#![doc = "..."]` attributes (which is how `syn` represents
  `//!` comments).
- **Diagnostic code:** `CTX004`.
- **Promotion path:** could move to Tier 2 if a `#[derive(Module)]` macro
  ever wraps file scaffolding. Until then, Tier 3b is the right home.
