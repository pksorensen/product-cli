# CTX005 — Function length

Every function body must be **at most 40 statement lines**, with a
soft target at 30. "Statement lines" means non-blank, non-brace-only
lines within the function body (matching the bash precursor's
heuristic).

> **Status:** shipped at `severity: warn` for one cycle. The bash
> precursor silently missed functions whose signatures spanned multiple
> lines, so this xtask check surfaces a backlog of pre-existing
> outliers. Promote to `deny` once the backlog is cleared. Tracked in
> [ADR-0005](../adr/ADR-0005-fitness-checks.md).

The check covers free functions (`fn foo() { ... }`), inherent and trait
impl methods (`impl Foo { fn bar() { ... } }`), and trait default methods
(`trait Foo { fn baz() { ... } }`).

## Why

A 40-line cap on functions, like the 400-line cap on files, forces a
refactor at the moment a function starts to do too many things. It also
keeps unit-test surface area sane — a function you can read on a single
screen is one you can test on a single screen.

See [ADR-0005](../adr/ADR-0005-fitness-checks.md).

## How to fix

In rough order of preference:

1. **Extract a helper.** If the function has a clear "do step A, then
   step B" structure, A and B are usually their own functions.
2. **Split along a pure-vs-effectful seam.** Per the Slice + Adapter
   pattern, separate `plan_*` (pure) from `apply_*` (I/O) and have the
   long function call both.
3. **Replace a chain of `match` arms with a lookup table.** If the
   function is mostly `match x { A => ..., B => ..., C => ... }` and each
   arm is structural, consider a `fn dispatch(x: &X) -> Y` over a const
   table of `(predicate, handler)` pairs.
4. **Remove dead branches.** Long functions sometimes accumulate
   defensive branches for cases that can't happen. Check whether the
   surrounding type system makes them unreachable.

## Examples

**Triggers CTX005:**

```rust
fn handle_request(req: Request) -> Response {
    // 45 lines of branching, validation, dispatch, error mapping...
}
```

**Passes CTX005:**

```rust
fn handle_request(req: Request) -> Response {
    let plan = plan_request(&req)?;
    apply_request(plan)
}
```

## Enforcement

- **Tier:** 3b (xtask + syn).
- **Implementation:** `xtask/src/checks/ctx005_function_length.rs`.
- **AST inspection:** parses each file with `syn`, visits every `ItemFn`
  / `ImplItemFn` / `TraitItemFn`, and counts statement lines in the body
  span using `proc_macro2`'s line numbers.
- **Diagnostic code:** `CTX005`.
- **Promotion path:** Tier 1/2 promotion would require a function-shape
  type-encoding (e.g. effectful builder traits) that the codebase
  doesn't yet have. Stays at Tier 3b.
